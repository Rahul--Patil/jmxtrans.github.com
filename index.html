JmxTrans Project

JMX made easy

JmxTrans is effectively the missing connector between speaking to a JVM via JMX on one end and whatever logging / monitoring / graphing package that you can dream up on the other end.

JmxTrans Project

JmxTrans is the perfect hub to collect JMX metrics from java servers (Tomcat, JBoss, etc)and export them to logging / monitoring / graphing systems.

The JmxTrans project is available at https://github.com/lookfirst/jmxtrans.

Embedded JmxTrans Project

Embedded JmxTrans was designed for cloud environements where the java servers topology is elastic and difficult to discover. In such case, the most easy thing is to get the java server (Tomcat, JBoss, etc) directly push its JMX metrics to the logging / monitoring / systems.

Embedded JmxTrans is also suited for batch and ephemeral java process thanks to the ease to export final JMX metrics at the end of the life of the process.

The Embedded JmxTrans project is available at https://github.com/jmxtrans/embedded-jmxtrans.
